# 装备、卡槽箱与工具包

## 玩家装备

盔甲、副手、Curios 饰品和装饰盔甲面板显示玩家的真实装备槽，不是复制品。

- 盔甲和副手无需附属模组。
- Curios 饰品栏需要 Curios API 和饰品栏卡。
- 装饰盔甲需要 Cosmetic Armor Reworked 和装饰盔甲卡。

这些槽位拒绝 Shift 快速移动和整理模组自动搬运，只允许手动放入或拿取，防止排序操作卸下装备。

当终端本体位于 Curios 槽并正在承载当前打开菜单时，那个**确切的终端实例**不能从饰品槽取下。关闭菜单后可正常取下；其它相同终端和其它饰品不会被锁定。

## 卡槽箱

卡槽箱用于保存 WCWT 支持的升级卡。将卡手动放入对应槽位；卡片是否可用仍取决于附属模组是否加载。移除提供当前面板的卡片后，相应面板会关闭或隐藏，但真实槽位内容会由服务器保持一致。

## 工具包

工具包用于保存不可堆叠的工具类物品。前 11 格为固定专用工具槽，后续通用槽可使用记忆功能。

1. 打开工具包面板，把工具手动放入通用槽。
2. 点击记忆按钮进入开启状态，再点击希望记住的通用槽。
3. 已记忆空槽会显示开启/悬停样式，并记录所对应工具类型。
4. 以后 Shift 移动匹配工具时，会优先进入已记忆且为空的槽。
5. 关闭记忆模式后，按钮恢复关闭样式，不会删除已保存的槽位记忆。

记忆按钮有关闭、悬停、开启三种交互状态，其中悬停与开启共用高亮样式。记忆槽数量由服务端 `toolkitSlotCount` 控制；更改数量不会让客户端自行突破服务器限制。

工具包不会复制物品。目标槽被占用、物品可堆叠或不符合工具条件时，会使用常规移动规则或拒绝放入。
